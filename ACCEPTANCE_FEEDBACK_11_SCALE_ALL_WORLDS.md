## 一句话结论

## 三、关键洞察：三种状态，指向同一个解法

把三个世界并排看，你的直觉就被数据证明了：

1. **shimmering-spheres（worldScale=1）** = 人物**太大**、世界被挤 → 「小气」。
2. **polka-dot（worldScale=3）** = 世界**够大够恢弘**✓，但人物**太小**、成金团 → 用力过猛。
3. **bright-gallery（世界天生大）** = 空间和人物**刚好** → demo 级。

→ **正确答案在 1 和 2 之间。** 你要的「大世界、小人物」不是玄学，就是给每个世界一个**适中的 `worldScale`（经验值约 1.5~2.2）**，让人物从「顶天」缩到「占画面高度的 1/2~2/3」，同时空间纵深打开。polka 的 3 太多、spheres 的 1 太少，bright 的比例是标杆。

## 四、可调的旋钮（已读代码确认，供 Claude 精确下手）

`lib/museum3d.js`：

1. **世界大小** = `world.worldScale`（config/worlds.js）。目前只有 polka 设了 3，其余 9 个都没设（=1）。
   - `scaleProfile()`（museum3d.js:11）会把 spawn/groundY/bounds 一起按 k 缩放，**等比放大整个世界，人物尺寸不变** → 正是你要的「世界变大、人显小」，且不破坏比例。
2. **人物身高** = 固定 1.75m（museum3d.js:461 `avatar.scale.setScalar(1.75/size.y)`）+ 相机眼高 `EYE_HEIGHT=1.6`（:6）。**不建议改人物**——改世界更干净。
3. **人物离相机的距离/间距** = 相机相对固定槽位（museum3d.js:477）：`back -1.6~-2.3m、side ±1.1m`。**这就是「人物永远贴脸顶天」的直接原因**——无论世界多大，三人始终锁在相机前 1.6m。
   - 想让人物显小又不失辨识度：把 `back` 拉远（如 -2.5~-3.5m）+ `side` 略加宽，比单纯狂调 worldScale 更可控。polka 之所以变金团，就是 worldScale=3 把这个近距槽位也放大了、人物被推太远。

## 五、建议（按性价比排序）

1. **[P0] 给每个「进 demo 的世界」单独调 `worldScale`**（约 1.5~2.2），以 bright-gallery 的比例为标杆。别用全局一个值——每个世界的原生尺度不同（见 metric.scale：spheres 1.5、van-gogh 1.5、floral 1.05…）。
2. **[P0] 调同行者槽位**（museum3d.js:477）把 `back` 从 -1.6/-2.3 拉到约 -2.5~-3.2m，让人物从「顶天贴脸」退到「半身/大半身在画面里」，露出更多空间。**这一步可能比 worldScale 更直接解决「小气」**——因为小气感主要来自人物贴脸，而非世界不够大。
3. **[P1] 修陷地/贴地**（van-gogh、floral）：这些世界人物半身陷进地面，是 `profile.groundY` 对不上真实地面 → 已有 `groundAt()` raycast（museum3d.js:341），应对每个同行者落脚点用它取地形高度，而不是套 profile 常量。
4. **[P1] demo 只用达标世界**：目前只有 **bright-gallery** 全绿、**shimmering-spheres** 调完比例后可作备选。其余（van-gogh/floral/conservatory/水园/海岸…）环境糊黑+落位崩，**不要放进主评审路径**，直到重新生成或调好。
5. **[P2] 圆点屋**：worldScale 从 3 降到约 2，并给同行者独立打光层（脱离金色环境光），可从「金团」救回人形。

## 六、直接回应你的判断

- ✅ **「别的场景挺糟糕」** —— 属实。除 bright-gallery 外，我看的 5 个世界都有明显问题（人物顶天 / 陷地 / 环境糊黑 / 雾状不可用）。
- ✅ **「世界大一点、人物小一点会更 make sense」** —— **完全正确，且是对的技术方向。** 代码已有 `worldScale` 支持等比放大世界而不动人物；polka 用它拿到了恢弘空间（只是数值过头）。落地做法 = 每世界调适中 worldScale（~1.5~2.2）+ 把同行者槽位从贴脸(-1.6m)拉远(~-3m)，以 bright-gallery 为标杆。
