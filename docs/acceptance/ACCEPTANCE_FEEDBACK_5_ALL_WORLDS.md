## 2. 关键规律（根因判断）

### 规律 A：mesh 世界比例对，splat 世界比例普遍崩
- **两个 mesh 世界（#1 圆点屋、#4 流光球境）环境清晰、人物贴地尺度对**（#1 本轮被扇形改动破坏，但环境本身是好的）。
- **七个 splat 世界**里，环境几乎都渲染成漆黑/棕雾/糊面（#2/#3/#5/#7/#8 环境基本看不见，#6/#9 半糊），而且人物落位大量出错（陷地、巨大、消失、背对）。
- 原因推测：splat 路径按 `metric.scale/ty` 手工复现根矩阵，但同行者/相机的落脚点、朝向、尺度是**按默认世界（或平地板 y=0）硬编的**，没有按每个世界不同的 `profile.groundY / scale / bounds / spawn` 去放。scale 越大、groundY 越极端（如 #7 groundY=1.1、#8 groundY=5.6、#5 scale=2.49）崩得越狠。

### 规律 B：open 世界（recommended:false）人物被巨大空间吞没
- #5/#7/#8 这类 bounds ±30~75m 的大世界，人物要么变成远处小点、要么因落位错误贴脸巨大，**开放世界不适合放三个近景同行者**。你在 config 里已把它们标 `recommended:false`，符合观察。

### 规律 C：splat 环境「上半屏黑 / 全黑」是普遍问题
- 多个 splat 世界上半部（天空/远景方向）完全没有 splat 数据，渲染成黑。可能是相机朝向（yaw）背对了世界主体，或 splat 只覆盖下半球。

## 3. 建议（按优先级）

1. **回滚扇形跟随**（P0）—— 默认圆点屋本轮从「能跟随」退化到「走动消失」。先回到上一轮「身前近景 + 相机偏移跟随」，再单独修第三人 Monet 的偏移，别用把人推远的扇形。
2. **同行者落位改为按世界 profile 驱动**（P0）—— 脚点 y 用该世界 `profile.groundY`（或对该世界 collider/mesh 向下 raycast），生成位置用 `profile.spawn` + `bounds` 约束，尺度按该世界实际 metric 归一，而不是套默认世界的常量。这是「换场景就崩」的根因。
3. **统一同行者朝向**（P1）—— 多个世界里人物背对相机；应让他们面朝相机或面朝行进方向，别背对访客。
4. **splat 环境上半屏黑 / 糊**（P1，资产层）—— 检查每个 splat 世界的 yaw 是否朝向主体、splat 是否只覆盖局部；#2/#3/#5/#7/#8 环境基本不可用。
5. **demo 世界收敛到 2 个 mesh**（建议）—— 现阶段真正达标的是 **#4 流光球境** 和 **#1 圆点屋**（修好跟随后）。建议 demo 主打这两个 mesh 世界，splat 世界作为「可选/实验」，别放进主评审路径。
6. open 世界（#5~#9）暂不进 demo，符合你已有的 `recommended:false`。

## 4. 亮点确认
- **#4 流光球境（mesh）是全场最佳**：清晰粉色球体隧道、三人全身、尺度一致、贴地、朝前。可作为「比例正确」的黄金参照，把它的 profile/落位逻辑套到其他世界。
- #9 阳光宫苑、#6 墨西哥庭院的**人物落位其实是对的**（三人全身、贴地、朝前、尺度齐），只是环境 splat 糊 —— 说明「人物落位」在部分 splat 世界能对，关键是逐世界 profile 而非全局常量。
